## What Token-2022 is

Token-2022 (also called Token Extensions) is an upgraded Solana token standard that supports features not possible with the original SPL token program. It was developed by Solana Labs and deployed to mainnet in 2024.

The key difference: Token-2022 tokens live in a different on-chain program than standard SPL tokens. They are a different type — not an upgrade of existing tokens.

## Token-2022 vs SPL: the core tradeoff

| | Standard SPL | Token-2022 |
|-|-------------|------------|
| DEX compatibility | Universal | Varies by DEX |
| Wallet support | Universal | Most modern wallets |
| Transfer fees | Not supported | Supported |
| Confidential transfers | Not supported | Supported |
| Interest-bearing | Not supported | Supported |
| Complexity | Low | Higher |
| Best for | Most tokens | Specific use cases |

**The default choice is SPL.** Choose Token-2022 only if you need a specific extension.

## Available extensions

### Transfer fee
Automatically deduct a percentage from every transfer. Use cases: protocol-level revenue, automatic burns, redistribution to holders. The fee percentage is set at creation and can be modified by the fee config authority.

### Confidential transfers
Hide transfer amounts while keeping transfers verifiable on-chain. Use cases: privacy-focused tokens, institutional use cases where amount visibility is a concern.

### Interest-bearing
The token's displayed balance increases over time based on an interest rate. Useful for yield-bearing tokens or bonds.

### Non-transferable
Tokens that cannot be transferred after minting. Suitable for soulbound tokens, certificates, or reputation tokens.

### Permanent delegate
A designated wallet that can transfer or burn any holder's tokens. Rarely appropriate for public tokens — use only with full understanding and disclosure.

### Metadata extension
Store metadata directly in the token's mint account (no separate Metaplex account). Simpler metadata management, though less battle-tested.

## When to use Token-2022

Use Token-2022 if:
- You need on-chain transfer fees
- You need confidential transfers
- You are building a yield product and want interest-bearing semantics
- Your specific DEX or protocol explicitly supports and recommends it

Use standard SPL if:
- You are creating a standard meme token or community token
- You need broad DEX and wallet compatibility
- You are unsure which to choose

<Warning>
**Token-2022 compatibility is not universal.** Some DEXs, wallets, and explorers may not fully support all Token-2022 extensions. Verify compatibility with your target ecosystem before choosing this standard.
</Warning>
